# Presentations du projet

Au cours du bloc, vous serez amenes a presenter votre projet sous differents formats. Ces presentations permettent de developper vos competences en communication et de valoriser votre travail aupres de differents publics.

**Les presentations ont lieu chaque vendredi.**

---

## Presentation 1 - Poster scientifique

| Element | Description |
|---------|-------------|
| Format | A1 ou A0, orientation portrait |
| Contenu | Titre accrocheur, problematique, solution proposee, maquettes, architecture envisagee |
| Langue | Francais |
| Objectif | Presenter la vision globale du projet avant le developpement |

### Conseils de realisation

- Utiliser une hierarchie visuelle claire (titres, sous-titres, corps de texte)
- Limiter le texte : privilegier les schemas, diagrammes et visuels
- Choisir une palette de couleurs coherente (3 couleurs maximum)
- Inclure un QR code vers le repository ou la maquette Figma
- Laisser des espaces blancs pour la lisibilite

### Contenu attendu

1. Titre du projet et logo/identite visuelle
2. Problematique adressee
3. Personas et utilisateurs cibles
4. Maquettes des pages principales
5. Architecture technique envisagee (schema MVC)
6. Planning previsionnel (diagramme de Gantt simplifie)

## Presentation 2 - Presentation classique (diaporama)

| Element | Description |
|---------|-------------|
| Format | PowerPoint / Google Slides / Beamer |
| Duree | 10 minutes + 5 minutes de questions |
| Contenu | Avancement du projet, demonstration des premieres pages, difficultes rencontrees |
| Langue | Francais |
| Objectif | Faire un point d'etape sur le developpement front-end |

### Structure recommandee

1. Rappel du contexte et des objectifs (1 slide)
2. Avancement par rapport au planning (1 slide)
3. Demonstration des pages HTML/CSS realisees (3-4 minutes)
4. Choix techniques effectues (CSS, architecture des fichiers) (2 slides)
5. Difficultes rencontrees et solutions (1 slide)
6. Prochaines etapes (1 slide)

### Criteres d'evaluation

- Clarte de la presentation
- Qualite des visuels (pas de mur de texte)
- Maitrise du sujet lors des questions
- Respect du temps imparti

## Presentation 3 - Pitch presentation (English)

| Element | Description |
|---------|-------------|
| Format | Oral presentation without slides (or 3 slides maximum) |
| Duration | 3 minutes strict |
| Content | Problem, solution, target users, key features, competitive advantage |
| Language | English |
| Objective | Practice concise communication and technical English |

### Structure (Elevator Pitch format)

1. **Hook** (15 sec): Start with a striking fact or question about international mobility
2. **Problem** (30 sec): Describe the current pain points for students seeking mobility
3. **Solution** (60 sec): Present your platform and its key features
4. **Value proposition** (45 sec): Explain what makes your solution unique
5. **Call to action** (30 sec): What do you want from your audience?

### Tips for success

- Practice until you can deliver it naturally without notes
- Use simple vocabulary and short sentences
- Maintain eye contact with the audience
- Speak clearly and at a moderate pace
- Time yourself during rehearsals

### Vocabulary help

| Francais | English |
|----------|---------|
| Mobilite internationale | International mobility / Study abroad |
| Universite partenaire | Partner university |
| Candidature | Application |
| Semestre d'echange | Exchange semester |
| Double diplome | Dual degree |

## Presentation 4 - Piece de theatre

| Element | Description |
|---------|-------------|
| Format | Saynete de 5 a 8 minutes |
| Participants | Tous les membres du groupe |
| Contenu | Mise en scene d'un scenario d'utilisation de la plateforme |
| Langue | Francais |
| Objectif | Presenter le projet de maniere creative et demontrer les fonctionnalites |

### Comment realiser une piece de theatre technique

#### 1. Ecriture du scenario

- Identifier 2-3 personas (ex: etudiant perdu, responsable RI deborde, ancien etudiant temoin)
- Creer une situation probleme initiale (recherche chaotique de mobilite)
- Introduire la solution (votre plateforme) comme element de resolution
- Conclure sur les benefices concrets

#### 2. Repartition des roles

| Role | Description |
|------|-------------|
| Narrateur (optionnel) | Contextualise les scenes et assure les transitions |
| Utilisateurs | Jouent les personas definis (etudiant, responsable RI, etc.) |
| Demonstrateur | Manipule l'application en temps reel sur l'ecran projete |

#### 3. Mise en scene

- Utiliser des accessoires simples (ordinateur portable, telephone, dossiers papier pour le "avant")
- Prevoir des transitions fluides entre les scenes
- Integrer la demonstration live de l'application dans le scenario
- Repeter plusieurs fois pour maitriser le timing

#### 4. Exemple de structure

| Scene | Description | Duree |
|-------|-------------|-------|
| Scene 1 | Un etudiant cherche desesperement des infos sur les mobilites (ton comique) | 1 min |
| Scene 2 | Le responsable RI croule sous les emails et dossiers papier | 1 min |
| Scene 3 | Decouverte de la plateforme, demonstration des fonctionnalites principales | 4 min |
| Scene 4 | Epilogue - temoignage d'un ancien etudiant satisfait | 1 min |

### Criteres d'evaluation

- Originalite et creativite du scenario
- Integration fluide de la demonstration technique
- Participation equilibree de tous les membres
- Qualite de la mise en scene et des transitions

## Presentation 5 - Demo live commentee (English)

| Element | Description |
|---------|-------------|
| Format | Live demonstration with technical commentary |
| Duration | 8-10 minutes |
| Content | Full user journey through the application |
| Language | English |
| Objective | Final presentation demonstrating the complete application |

### Scenario structure

| Step | Feature to demonstrate | Duration |
|------|------------------------|----------|
| 1 | Homepage and navigation overview | 1 min |
| 2 | Student registration and profile creation | 1.5 min |
| 3 | Searching for mobility opportunities (filters, sorting) | 2 min |
| 4 | Viewing university partner details and reviews | 1.5 min |
| 5 | Application submission process | 2 min |
| 6 | Admin/RI dashboard demonstration | 2 min |

### Technical vocabulary

| Francais | English |
|----------|---------|
| Se connecter | To log in |
| S'inscrire | To register / To sign up |
| Rechercher | To search |
| Filtrer | To filter |
| Postuler | To apply |
| Tableau de bord | Dashboard |
| Base de donnees | Database |
| Formulaire | Form |

### Tips for the live demo

- Prepare a backup plan (screenshots, video) in case of technical issues
- Pre-fill some data to avoid typing during the demo
- Have multiple browser tabs ready for different user roles
- Practice the demo multiple times to ensure smooth transitions
- Speak slowly and clearly - the audience needs to follow both the screen and your commentary

### Evaluation criteria

- Fluency in English
- Completeness of the demonstration
- Ability to handle questions in English
- Technical quality of the application demonstrated
