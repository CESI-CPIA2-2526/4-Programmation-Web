# Boucle PBL : Dynamiser côté front

**Type :** Prosit / Workshop — **Thème :** JavaScript, DOM, événements, validation de formulaires

**Votre mission :** Dynamiser l'interface front-end de Web4all en utilisant JavaScript : réagir aux interactions de l'utilisateur, manipuler le DOM, valider les formulaires côté client et respecter les bonnes pratiques de codage.

**Livrables :**

| Livrable | Ce qu'on attend |
|----------|-----------------|
| Intégration JS | Fichier JavaScript correctement lié à la page HTML |
| Manipulation DOM | Modification dynamique du contenu et des styles via JS |
| Gestion d'événements | Réactions aux actions utilisateur (clics, saisies…) |
| Validation de formulaire | Contrôle des saisies côté client avant envoi |

## Sujet : Énoncé

### Oh my code, je parle le JavaScript

*Ce matin dans les locaux de Web4all...*

**Jean-Marc :**

> « Bonjour à tous. La structure PHP tourne, les données remontent de la base. Maintenant on s'attaque au front. Les utilisateurs se plaignent que les pages manquent de dynamisme. Léa, c'est toi qui prends le JS ? »

**Léa :**

> « Avec plaisir. On part sur du JavaScript vanilla ou on intègre une lib ? »

**Estelle :**

> « On commence vanilla. Il faut d'abord maîtriser les bases : manipuler le DOM, réagir aux événements, valider les formulaires avant envoi. On verra jQuery si le temps le permet. »

**Léa :**

> « Et pour la validation, on duplique le travail fait côté serveur ? »

**Estelle :**

> « La validation côté client, c'est du confort utilisateur — on évite un aller-retour serveur pour une erreur de saisie évidente. Mais la validation serveur reste indispensable : ne jamais faire confiance au client. »

**Jean-Marc :**

> « Exactement. Priorité : relier les événements aux bons éléments du DOM, valider les formulaires, et maintenir un code propre et bien structuré. »

**Léa :**

> « Compris. »
